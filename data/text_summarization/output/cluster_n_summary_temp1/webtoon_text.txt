<subject>독자와 웹툰 간 로그데이터를 활용한 웹툰 추천 시스템</subject>
<team>강나영, 김나현, 엄성원, 이철민</team>
<index>프로젝트 소개, 프로젝트 배경, 프로젝트 목표, 데이터 수집 및 전처리, 데이터 소스 설명, 데이터 전처리, 모델 선택 및 학습, 모델 비교, 최종 모델 선정, 웹툰 추천 시스템 구현, 기존 사용자, 신규 사용자, 결론 및 향후 과제, 프로젝트 요약 및 의의, 한계점 및 향후 과제</index>
<main>프로젝트 소개</main>
<subtitle>프로젝트 배경</subtitle>
<content>BITAmir 프로젝트는 정제된 대량의 데이터를 활용하여 모델 선택 범위를 넓히고, 사용자의 데이터를 효과적으로 사용할 수 있는 주제를 선정하여 국내 최신 데이터를 기반으로 직관적인 추천 시스템의 작동을 확인하고자 합니다.</content>

<subtitle>프로젝트 목표</subtitle>
<content>이 프로젝트의 목표는 사용자 웹툰 데이터를 이용하여 개인화된 추천 시스템을 개발하고, 사용자가 흥미를 느낄 웹툰을 정확히 추천하며, 사용자와 아이템의 데이터에 적합한 알고리즘을 탐색하고 적용하는 것입니다.</content>
<main> 프로젝트 배경</main>
<subtitle>[데이터 수집 및 전처리]</subtitle>
<content>프로젝트에서는 여러 CSV 파일을 활용하여 애니메이션 및 웹툰 관련 데이터를 수집하고 전처리하였다. 주요 데이터 소스로는 고객 정보, 웹툰 기사 데이터, 사용자 상호작용 데이터가 포함되었다.</content>
<main> 프로젝트 목표</main>
<subtitle>[데이터 수집 및 전처리]</subtitle>
<content>프로젝트의 목표는 649만 개의 데이터 중 100개 이상을 평가한 유저만을 남기고, 피봇 테이블 형식으로 데이터를 전처리하여 유용한 인사이트를 추출하는 것입니다.</content>
<main> 데이터 수집 및 전처리</main>
<subtitle>데이터 수집</subtitle>
<content>사용자와 아이템 간의 상호작용 데이터를 수집하여 모델 학습에 필요한 정보를 확보하는 과정이다. 여기에는 사용자 평가 및 구매 정보가 포함된다.</content>

<subtitle>데이터 전처리</subtitle>
<content>수집된 데이터는 이상치 제거, 결측치 처리 및 정규화 등의 전처리 과정을 통해 모델 학습에 적합한 형태로 가공된다.</content>
<main> 데이터 소스 설명</main>
<subtitle>NCF (Neural Collaborative Filtering)</subtitle>
<content>NCF는 사용자와 아이템을 임베딩 벡터로 표현하여 사용자-아이템 간의 상호작용을 기반으로 추천 아이템의 확률을 학습하는 모델입니다. 이 모델은 복잡한 비선형 관계를 학습할 수 있는 신경망을 활용하고, 추천 알고리즘의 직관성과 사용성을 장점으로 삼지만, 데이터의 불균형(Long tail Data Sparsity) 문제가 단점으로 지적됩니다.</content>
<main> 데이터 전처리</main>
<subtitle>[데이터 전처리의 필요성]</subtitle>
<content>데이터 전처리는 머신러닝 모델에서 정확한 예측을 위해 필수적인 단계로, 데이터를 정제하고 변환하여 모델이 이해할 수 있도록 준비하는 과정이다.</content>

<subtitle>[데이터 정제 및 변환]</subtitle>
<content>데이터 정제는 결측값 처리, 이상치 제거 등으로 이루어지며, 데이터 변환은 특성 스케일링 및 인코딩 등이 포함되어 모델의 성능을 향상시키는 데 중요하다.</content>

<subtitle>[데이터 전처리의 효과]</subtitle>
<content>적절한 데이터 전처리는 모델 학습의 효율성을 높이고, 결과의 신뢰성을 높여 궁극적으로 예측 성능을 개선하는 데 기여한다.</content>
<main> 모델 선택 및 학습</main>
<subtitle>SVD 모델 개념</subtitle>
<content>사용자-아이템 행렬을 세 개의 행렬로 분해하여 사용자와 아이템의 잠재적 특성을 추출하는 방식으로, 이는 차원 축소와 계산 효율성을 증가시키며 노이즈를 제거하는 장점이 있다. 그러나 큰 데이터셋에서는 계산 비용이 높아지고 실시간 업데이트가 어려운 단점이 있다.</content>

<subtitle>SVD 모델링 및 결과출력</subtitle>
<content>모델 성능 평가는 RMSE를 이용하여 이루어지며, 예측 결과와 실제 값을 비교하여 모델의 예측 능력을 평가한다. 이 과정에서 Surprise 라이브러리를 활용하여 SVD 모델을 생성하고 성능을 평가한다.</content>
<main> 모델 비교</main>
<subtitle>모델 선택 및 학습</subtitle>
<content>모델 선택 과정에서 SVD, BITAmir 등 여러 모델이 고려되었으며, 이를 바탕으로 모델의 성능을 분석하였다.</content>

<subtitle>하이퍼파라미터 최적화</subtitle>
<content>GridSearchCV를 활용하여 각 모델의 하이퍼파라미터를 최적화하였고, 이를 통해 성능을 향상시킬 수 있었다.</content>

<subtitle>모델 성능 평가</subtitle>
<content>모델 재학습 후 RMSE 지표를 통해 최종 모델의 성능을 평가하였으며, 결과적으로 어느 모델이 가장 우수한지를 판별하였다.</content>
<main> 최종 모델 선정</main>
<subtitle>모델 선정 이유</subtitle>
<content>ALS(Alternating Least Squares) 모델이 선정된 이유는 추천 시스템에서 효과적인 행렬 분해 기법으로, 사용자와 아이템 간의 상호작용을 저차원으로 분해하여 잠재 요인을 학습할 수 있기 때문이다. 이 모델은 손실 함수 최적화와 정규화를 통해 대규모 데이터셋에서도 높은 성능을 유지한다.</content>
<main> 웹툰 추천 시스템 구현</main>
<subtitle>[최종 모델 선정]</subtitle>
<content>BITAmir 프로젝트에서 Alternating Least Squares(ALS) 모델을 선정한 이유는, 사용자의 호불호를 정확하게 파악하기 어려운 Implicit Feedback 데이터를 효과적으로 처리할 수 있기 때문이다. ALS는 대규모 데이터에 적합하며 RMSE 수치가 낮아 추천의 정확성을 높일 수 있다.</content>

<subtitle>[결론 및 향후 과제]</subtitle>
<content>프로젝트의 분석 결과, CF, NCF, SVD, ALS 등 다양한 추천 알고리즘 모델을 비교하여 최적의 모델을 선택함으로써 사용자 맞춤형 추천이 가능하다는 결론을 도출하였다. Implicit feedback을 활용한 ALS 모델은 웹툰 추천 시스템의 사용자 만족도와 플랫폼 이용률을 향상시킬 수 있는 좋은 방법임을 보여준다.</content>
<main> 기존 사용자</main>
<subtitle>[최종 모델 선정]</subtitle>
<content>기존 사용자에게 적합한 웹툰 추천을 위해 BITAmir와 ALS(Alternating Least Squares) 모델의 성능을 평가하고 최종 모델을 선정하였다.</content>

<subtitle>[모델 성능 평가]</subtitle>
<content>모델의 성능은 RMSE를 통해 측정되며, 이를 바탕으로 기존 사용자에게 더욱 효과적인 추천 시스템을 구축하는 것이 목표이다.</content>
<main> 신규 사용자</main>
<subtitle>[신규 사용자 추천]</subtitle>
<content>신규 사용자 추천 시스템에서 추천 결과를 사용자 ID에 따라 제공하며, 추천된 웹툰은 사용자의 선호 장르와 유형에 기반하여 달라질 수 있다. 신규 사용자는 이전의 웹툰 시청 기록이 없기 때문에 초기 추천 알고리즘이 중요하다.</content>
<main> 결론 및 향후 과제</main>
<subtitle>결론</subtitle>
<content>웹툰 추천 시스템은 신규 사용자의 선호도를 기반으로 하여 맞춤형 추천 결과를 제공하며, 이를 통해 사용자의 만족도를 높일 수 있다.</content>

<subtitle>향후 과제</subtitle>
<content>향후에는 더 많은 사용자 데이터를 수집하고, 알고리즘을 개선하여 추천의 정확성을 높이는 과제가 필요하다.</content>
<main> 프로젝트 요약 및 의의</main>
<subtitle>결론 및 향후 과제</subtitle>
<content>프로젝트의 결론으로, 이미 정제된 데이터를 사용했지만 사용자와 아이템 간의 상호작용 등급이 부족하여 사용자의 선호도를 정확하게 파악하기 어려웠음을 지적하고 있다. Cold Start Problem 도 언급하며, 신규 사용자에 대한 추천 시스템의 한계를 강조하고 있다. 향후 과제로는 명시적 데이터 수집을 통한 모델 성능 향상과 아이템 데이터 내 장르 정보를 활용한 모델 개선이 제안된다.</content>
<main> 한계점 및 향후 과제</main>
<subtitle>제한된 데이터</subtitle>
<content>프로젝트에 사용된 데이터의 양과 질이 제한적이며, 이는 추천 시스템의 정확성과 효율성을 저하시킬 수 있다.</content>

<subtitle>알고리즘 최적화 필요성</subtitle>
<content>현재 사용된 알고리즘은 더 발전할 여지가 있으며, 향후 더 나은 성능을 위해 다양한 알고리즘의 시험과 비교가 필요하다.</content>

<subtitle>사용자 피드백 부족</subtitle>
<content>사용자 피드백을 효과적으로 수집하고 분석하는 시스템이 부족하여, 추천 시스템의 개선에 필요한 데이터가 충분하지 않다.</content>

<subtitle>확장성 문제</subtitle>
<content>시스템의 확장성이 검토되지 않아, 사용자 수의 증가나 데이터 양의 증가에 대한 대비가 필요하다.</content>

<subtitle>향후 연구 방향</subtitle>
<content>향후 사용자 맞춤형 추천 시스템 개발 및 다양한 데이터를 통합하는 방향으로 연구가 진행되어야 한다.</content>
