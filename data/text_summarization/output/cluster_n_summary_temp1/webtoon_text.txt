<main>프로젝트 소개</main>
<sub>프로젝트 배경</sub>
<content>BITAmir 프로젝트는 정제된 대량의 데이터를 활용하여 모델 선택의 범위를 넓히고, 사용자의 데이터를 효과적으로 사용할 수 있는 주제를 선정하는 것을 목표로 합니다. 특히, 국내 최신 데이터를 활용하여 직관적으로 추천 시스템이 작동하는 방식을 검증하고자 합니다.</content>

<sub>프로젝트 목표</sub>
<content>이 프로젝트의 목표는 사용자 데이터와 웹툰 데이터를 이용해 개인화된 추천 시스템을 개발하는 것이며, 사용자에게 흥미를 느낄 만한 웹툰을 정확히 추천하는데 중점을 두고 있습니다. 이를 위해 사용자와 아이템의 데이터에 적합한 알고리즘을 탐색하고 적용할 계획입니다.</content>
<main> 프로젝트 배경</main>
<sub>데이터 수집 및 전처리</sub>
<content>본 프로젝트에서는 웹툰 관련 데이터를 수집하기 위해 여러 데이터 소스(Webtoon_CustomersCsV, Webtoon_articlescs!, Webtoon_interaction_traincs!)를 사용하였으며, 이를 통해 사용자 상호작용 데이터를 전처리하는 과정이 포함된다.</content>
<main> 프로젝트 목표</main>
<sub>데이터 수집 및 전처리</sub>
<content>649만 개의 데이터를 수집하고, 100개 이상의 평가를 한 유저만을 필터링하여 남기는 것을 목표로 하며, 피봇 테이블 형식으로 데이터를 구조화하는 과정이 포함된다.</content>
<main> 데이터 수집 및 전처리</main>
<sub>[데이터 전처리]</sub>
<content>데이터 전처리는 사용자 및 아이템 간의 유사도를 기반으로 추천 알고리즘을 설계하는 과정에서 필수적이다. 사용자 간 유사성(User-Based) 혹은 아이템 간 유사성(Item-Based)을 측정하는 방식이 있으며, 각기 다른 데이터 크기와 정보 수준에 따라 적합한 접근방식을 선택할 수 있다.</content>

<sub>[추천 알고리즘의 적용]</sub>
<content>추천 알고리즘은 데이터 전처리가 완료된 후 유사한 취향을 가진 사용자들 간의 상관관계를 이용하여 추천을 생성한다. 직관적인 결과를 제공하지만 Cold Start 문제와 높은 계산량으로 인해 추천 효율성이 떨어질 수 있다는 단점을 가지고 있다.</content>
<main> 데이터 소스 설명</main>
<sub>모델 선택 및 학습 NCF</sub>
<content>NCF(Neural Collaborative Filtering) 모델은 사용자와 아이템을 각각 임베딩 벡터로 표현하고, 이들의 상호작용을 통해 학습하여 아이템 추천의 확률을 계산한다. 이 모델은 복잡한 비선형 관계를 학습할 수 있어 추천 알고리즘의 직관성과 사용성을 강화하지만, Long tail 데이터와 데이터의 희소성 문제가 단점으로 지적된다.</content>
<main> 데이터 전처리</main>
<sub>[모델 선택]</sub>
<content>데이터 전처리 과정에서 적합한 모델을 선택하는 것이 중요하며, NCF(Neural Collaborative Filtering)와 같은 알고리즘을 활용해 최적화된 결과를 도출할 수 있다.</content>

<sub>[학습]</sub>
<content>데이터 전처리를 통해 모델 학습의 효율을 높이고, 데이터의 품질을 향상시켜 결과적으로 더 나은 성능의 모델을 생성할 수 있다.</content>

<sub>[결과 출력]</sub>
<content>최종적으로 NCF 모델의 결과를 출력하기 위해서는 데이터 전처리를 통해 정제된 데이터를 바탕으로 정확하고 신뢰할 수 있는 결과를 제공해야 한다.</content>
<main> 모델 선택 및 학습</main>
<sub>SVD(Singular Value Decomposition) 개념</sub>
<content>SVD는 사용자-아이템 행렬을 세 개의 하위 행렬로 분해하여 사용자와 아이템의 잠재적 특성을 추출하는 방법이다. 이 접근법은 차원 축소를 통해 계산 효율성을 높이고 노이즈를 제거하는 장점이 있지만, 큰 데이터셋에서는 계산 비용이 증가하고 실시간 업데이트가 어려운 단점이 있다.</content>

<sub>모델링 및 결과 출력</sub>
<content>모델 성능은 RMSE(평균 제곱근 오차)로 평가되며, SVD 모델을 생성하기 위해 Surprise와 같은 라이브러리를 사용할 수 있다. 예측 결과는 주어진 테스트 세트와 비교하여 정확성을 측정한다.</content>
<main> 모델 비교</main>
<sub>SVD</sub>
<content>SVD 모델링에 대한 내용을 다루며, 결과로 Value 출력과 Decomposition을 활용하여 모델을 분석함.</content>

<sub>BITAmir</sub>
<content>BITAmir 모델의 특성 및 작동 방식을 설명하며, 모델 비교 시의 장점을 언급함.</content>

<sub>SVDSingular</sub>
<content>SVDSingular 모델의 특징과 성능에 대해 논의하고, 다른 모델들과의 비교를 통해 경쟁력을 평가함.</content>

<sub>하이퍼파라미터 최적화</sub>
<content>GridSearchCV를 통해 각 모델의 하이퍼파라미터를 최적화하며, 다양한 모델 간 성능 차이를 살펴봄.</content>

<sub>모델 성능 평가</sub>
<content>모델 재학습 후 RMSE를 기준으로 성능을 평가하고, 이를 통해 각 모델의 유효성을 비교함.</content>
<main> 최종 모델 선정</main>
<sub>[모델 선정 이유]</sub>
<content>최종 모델로 ALS(Alternating Least Squares)를 선정한 이유는 추천 시스템에 적합한 행렬 분해 기법으로, 대규모 데이터셋에서 뛰어난 성능을 보이며, 손실 함수 최적화와 정규화 기능이 우수하기 때문이다.</content>
<main> 웹툰 추천 시스템 구현</main>
<sub>[최종 모델 선정]</sub>
<content>ALS 모델이 선택된 이유는 사용자 호불호 파악의 어려움과 대규모 데이터 적합성 때문이며, 임플리시트 피드백을 효과적으로 처리할 수 있는 능력 덕분이다.</content>

<sub>[결론 및 향후 과제]</sub>
<content>여러 추천 알고리즘을 비교 분석한 결과 ALS 모델을 통해 최적의 추천 시스템을 구축해 사용자 만족도를 높이고 플랫폼 이용률을 증가시킬 수 있는 방향성을 제시하고 있다.</content>
<main> 기존 사용자</main>
<sub>[모델 성능 평가]</sub>
<content>기존 사용자에 대한 모델 성능 평가는 RMSE(평균 제곱근 오차)를 통해 진행되며, 이는 추천 시스템의 정확성을 나타내는 중요한 지표이다.</content>

<sub>[웹툰 추천 진행]</sub>
<content>기존 사용자를 대상으로 한 웹툰 추천은 ALS(Alternating Least Squares) 모델링을 통해 이루어지며, 사용자의 선호도를 반영한 맞춤형 추천 결과를 출력한다.</content>

<main> 결론 및 향후 과제</main>
<sub>결론</sub>
<content>웹툰 추천 시스템은 신규 사용자의 선호 웹툰을 기반으로 하여 추천 결과를 제시하며, RMSE 값은 추천의 정확성을 나타낸다.</content>

<sub>향후 과제</sub>
<content>향후 개선 과제로는 추천 정확도를 높이기 위해 더 많은 사용자 데이터를 수집하고, 알고리즘을 최적화하는 것이 필요하다.</content>
<main> 프로젝트 요약 및 의의</main>
<sub>결론 및 향후 과제</sub>
<content>프로젝트의 결론은 이미 정제된 데이터를 사용했지만, 사용자와 아이템 간의 상호작용 평점이 없어 선호도를 정확히 파악하기 어려웠다는 것이다. 향후 과제로는 사용자 피드백을 통한 명시적 데이터 수집과 아이템 데이터의 장르 컬럼 활용을 통해 모델의 성능을 개선하는 방안을 제시하고 있다.</content>

<sub>Cold Start Problem</sub>
<content>추천 시스템이 사용자의 이전 행동이 없이 작동하기 때문에 신규 사용자에 대한 추천에 한계가 있으며, 이는 'Cold Start Problem'이라는 문제로 인해 발생한다.</content>
