<main>프로젝트 소개</main>
<sub>프로젝트 배경</sub> <content>정제된 '대량의 데이터'를 사용하여 모델 선택 범위를 넓히고자 하며, 사용자 데이터를 효과적으로 사용할 수 있는 주제 선정과 직관적인 추천 시스템의 작동을 확인할 수 있도록 가급적 국내 최신 데이터를 활용합니다.</content> <page>1</page>
<sub>프로젝트 목표</sub> <content>사용자 데이터와 웹툰 데이터를 이용해 개인화된 추천 시스템을 개발하여 사용자가 흥미를 느낄 만한 웹툰을 정확히 추천하고, 사용자와 아이템의 데이터에 맞는 알고리즘을 탐색 및 적용합니다.</content> <page>2</page>

<main>데이터 수집 및 전처리</main>
<sub>데이터 소스 설명</sub> <content>Webtoon_CustomersCsV, Webtoon_articlescs!, Webtoon_interaction_traincs! 등의 다양한 데이터 소스를 사용합니다.</content> <page>3</page>
<sub>데이터 전처리</sub> <content>649만 개의 데이터 중 100개 이상 평가한 유저만 남기며, 피봇 테이블 형식의 데이터로 변환합니다.</content> <page>4</page>

<main>모델 선택 및 학습</main>
<sub>Filtering 모델 CFCCollaborative</sub> <content>User-Based와 Item-Based 알고리즘을 통해 비슷한 취향을 가진 사용자들이 다른 아이템에 대해서도 비슷한 취향을 가지고 있을 것이라는 가정으로 추천을 진행하며, 직관적인 결과를 제공하나 Cold Start Problem이 존재합니다.</content> <page>5</page>
<sub>NCF Filtering 모델 Neura Collaborative</sub> <content>사용자와 아이템을 embedding vector로 표현하고, 사용자-아이템 간 interaction을 바탕으로 추천 확률을 학습하여 아이템을 추천합니다. 장점은 복잡한 비선형 학습이 가능하나 데이터의 희소성 문제를 안고 있습니다.</content> <page>6</page>
<sub>모델 SVDSingular Value Decomposition</sub> <content>사용자-아이템 행렬을 세 개의 행렬로 분해하여 잠재적 특성을 추출하며, 차원 축소를 통해 계산 효율성을 증가시키지만 실시간 업데이트가 어렵습니다.</content> <page>7</page>
<sub>ALS 모델 Alternating Least Squares</sub> <content>사용자-아이템 상호작용 행렬을 저차원 행렬로 분해하여 잠재 요인을 학습하는 방식으로, 대규모 데이터셋에 적합합니다.</content> <page>8</page>

<main>최종 모델 선정</main>
<sub>모델 선정 이유 ALS Alternating Least Squares</sub> <content>Implicit Feedback의 형태로 수집된 데이터에서 사용자 선호를 정확히 파악하기 어려운 문제를 해결하며, 대규모 데이터에 적합하여 RMSE 수치가 우수합니다.</content> <page>9</page>
<sub>ALS Alternating Least Squares 모델링 및 결과 출력</sub> <content>웹툰 추천 진행 모델 성능 평가를 통해 RMSE를 측정합니다.</content> <page>10</page>
