<main>03 모델 선택 및 학습</main>
<sub>Filtering 모델 CF (Collaborative)</sub> <content>개념: 비슷한 취향을 가진 사용자들이 다른 아이템에 대해서도 비슷한 취향을 가진다고 가정하고 추천. User-Based와 Item-Based로 나뉨. 특징: 데이터 크기가 작고 각 사용자에 대한 충분한 정보가 있는 경우에는 UBCF, 정보가 없는 경우에는 IBCF가 적합. 장점: 직관적인 결과, 상품의 정보 없이 추천 가능. 단점: Cold Start Problem, 계산량이 많아 추천의 효율성 떨어짐. 모델 성능 평가: RMSE</content> <page>7</page>
<sub>NCF Filtering 모델 (Neural Collaborative)</sub> <content>개념: 사용자와 아이템을 각각 embedding vector로 표현하고, interaction을 바탕으로 학습하여 추천. 장점: 추천 알고리즘의 직관성과 사용성. 단점: Long tail Data Sparsity.</content> <page>9</page>
<sub>SVD 모델 (Singular Value Decomposition)</sub> <content>개념: 사용자-아이템 행렬을 세 개의 행렬로 분해하여 잠재적 특성을 추출. 특징: 대각행렬을 통한 특이값 분해로 유저 아이템의 행과 열 개수가 다른 추천모델에도 적합. 장점: 차원 축소를 통해 계산 효율성 증가 및 노이즈 제거. 단점: 큰 데이터셋에서 계산 비용 높아짐, 실시간 업데이트 어려움. 모델 성능 평가: RMSE.</content> <page>11</page>
<sub>ALS 모델 (Alternating Least Squares)</sub> <content>개념: 추천 시스템에서 주로 사용되는 행렬 분해 기법. 특징: 교대 최소 제곱법, 손실 함수 최적화, 정규화, 대규모 데이터셋에 적합.</content> <page>14</page>

<main>03 최종 모델 선정</main>
<sub>모델 선정 이유 ALS (Alternating Least Squares)</sub> <content>Implicit Feedback의 형태로 수집된 데이터의 문제점: 사용자의 호불호를 정확히 파악하기 어려움. 선호와 비선호를 binary로 분류하는 방법 사용. 대규모 데이터에 적합. RMSE 수치: 0.1.</content> <page>16</page>
<sub>ALS 모델링 및 결과 출력</sub> <content>웹툰 추천 진행, 모델 성능 평가: RMSE.</content> <page>17</page>

<main>05 결론 및 향후 과제</main>
<sub>프로젝트 요약 및 의의</sub> <content>여러 가지 추천 알고리즘 모델을 구축하고 비교 분석하여 최적의 모델을 선택함으로써 사용자에게 더 나은 추천 제공. Explicit feedback 없이 Implicit feedback을 사용한 모델 구축으로 최상의 결과 도출. 사용자 선호에 맞는 웹툰 제공으로 만족도 향상 및 플랫폼 이용률 증가.</content> <page>20</page>
<sub>한계점 및 향후 과제</sub> <content>정제된 데이터 사용으로 사용자 선호도 파악 어려움. 신규 사용자에 대한 추천에는 Cold Start Problem 존재. 향후 과제: Explicit feedback 데이터 수집을 통한 모델 성능 향상, genre 컬럼을 활용한 모델 개선.</content> <page>21</page>
