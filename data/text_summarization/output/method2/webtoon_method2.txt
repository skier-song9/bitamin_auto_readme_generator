<main>프로젝트 소개</main>
<sub>프로젝트 배경</sub>
<content>사용자의 데이터를 효과적으로 활용하기 위해, 대량의 정제된 데이터를 사용하고 국내 최신 데이터를 활용하여 직관적인 추천 시스템을 구축하려는 목표를 가지고 있다.</content>
<sub>프로젝트 목표</sub>
<content>개인화된 웹툰 추천 시스템을 개발하여 사용자의 흥미를 유도하고, 데이터에 맞는 알고리즘을 탐색 및 적용하는 것이다.</content>

<main>02 데이터 수집 및 전처리</main>
<sub>데이터 소스 설명</sub>
<content>주요 데이터 소스로는 Webtoon_CustomersCsV, Webtoon_articlescs, Webtoon_interaction_traincs가 사용되었으며, interaction data는 pandas를 이용해 읽어들였다.</content>
<sub>데이터 전처리</sub>
<content>649만 개의 데이터 중 100개 이상 평가한 사용자만 남기고, 피봇 테이블 형식의 데이터를 생성했다.</content>

<main>03 모델 선택 및 학습</main>
<sub>CF (Collaborative Filtering)</sub>
<content>사용자 유사도를 측정하는 User-Based와 아이템 간 유사도를 측정하는 Item-Based로 나뉘며, 직관적인 결과를 제공하나 Cold Start Problem과 계산량이 많아 효율성이 떨어질 수 있다.</content>
<sub>NCF (Neural Collaborative Filtering)</sub>
<content>사용자와 아이템을 embedding vector로 표현하여 상호작용을 학습하며, 복잡한 비선형 학습이 가능하나 Long tail Data와 Data Sparsity 문제가 있다.</content>
<sub>SVD (Singular Value Decomposition)</sub>
<content>사용자-아이템 행렬을 세 개의 행렬로 분해하여 잠재적 특성을 추출하며, 차원 축소를 통해 계산 효율성을 증가시킬 수 있으나, 큰 데이터셋에서 계산 비용이 높아질 수 있다.</content>
<sub>ALS (Alternating Least Squares)</sub>
<content>사용자-아이템 상호작용 행렬을 두 개의 저차원 행렬로 분해하여 학습하며, 대규모 데이터셋에 적합하다.</content>

<main>03 최종 모델 선정</main>
<sub>ALS (Alternating Least Squares)</sub>
<content>Implicit Feedback 형태의 데이터를 기반으로 하여 대규모 데이터에 적합하며, RMSE 수치가 0.1인 ALS 모델을 최종 선택하였다.</content>

<main>04 웹툰 추천 시스템 구현</main>
<sub>기존 사용자</sub>
<content>기존 사용자의 이전 시청 데이터를 기반으로 웹툰을 추천했다.</content>
<sub>신규 사용자</sub>
<content>신규 사용자가 좋아하는 웹툰 3개를 입력하여 추천 결과를 도출했다.</content>

<main>05 결론 및 향후 과제</main>
<sub>프로젝트 요약 및 의의</sub>
<content>다양한 추천 알고리즘을 비교 분석한 결과, ALS 모델이 최적임을 확인하였으며, 이를 통해 사용자 맞춤형 웹툰 추천을 제공하여 만족도를 높였다.</content>
<sub>한계점 및 향후 과제</sub>
<content>명시적 데이터 수집이 부족하여 사용자 선호도를 정확히 파악하기 어려웠으며, Cold Start Problem을 해결하기 위한 명시적 feedback 데이터 수집 및 모델 개선이 필요하다.</content>
