<main>프로젝트 소개</main>
<sub>프로젝트 배경</sub>
<content>대량의 정제된 데이터를 사용해 모델 선택을 다양화하고, 사용자 데이터를 효과적으로 활용할 수 있는 주제를 선정하며, 직관적으로 추천 시스템 작동을 확인하기 위해 국내 최신 데이터를 활용하고자 함.</content>
<sub>프로젝트 목표</sub>
<content>사용자 데이터와 웹툰 데이터를 활용해 개인화된 추천 시스템을 개발, 사용자에게 맞춤형 웹툰을 정확히 추천. 데이터에 적합한 알고리즘을 탐색 및 적용.</content>

<main>데이터 수집 및 전처리</main>
<sub>데이터 소스 설명</sub>
<content>세 개의 CSV 파일(Webtoon_Customers.csv, Webtoon_articles.csv, Webtoon_interaction_train.csv)이 있으며, interaction data, webtoon data, user data는 각각 '/data/webtoon_interaction', '/data/webtoon', '/data/webtoon_Customers.csv' 경로에서 불러옵니다.</content>
<sub>데이터 전처리</sub>
<content>649만 개의 데이터에서 100개 이상 평가한 유저만 남겼음.</content>
<sub>데이터 전처리</sub>
<content>피봇 테이블 데이터 생성.</content>

<main>모델 선택 및 학습</main>
<sub>Filtering 모델 CF (Collaborative Filtering)</sub>
<content>비슷한 취향을 가진 사용자들이 다른 아이템에 대해서도 비슷한 취향을 가진다고 가정하여 추천하는 알고리즘으로, 사용자 유사도를 측정하는 User-Based와 아이템 간 유사도를 측정하는 Item-Based로 나뉩니다.</content>
<content>데이터가 작고 사용자 정보가 충분하면 UBCF, 데이터가 크거나 사용자 정보가 부족하면 IBCF가 적합합니다. 이 방법은 직관적이고 상품 정보 없이 추천이 가능하지만, Cold Start Problem과 계산량이 많아 효율성이 떨어질 수 있습니다.</content>
<sub>모델 성능 평가</sub>
<content>RMSE는 Root Mean Squared Error의 약자로, 예측 값과 실제 값 간의 차이를 측정하는 지표입니다. 작을수록 예측이 정확함을 의미합니다.</content>
<sub>NCF Filtering 모델 (Neural Collaborative Filtering)</sub>
<content>사용자와 아이템을 embedding vector로 표현하여, 사용자-아이템 간 interaction을 학습하고 확률 높은 순서로 아이템을 추천.</content>
<content>특징: 복잡한 비선형 학습
장점: 추천 알고리즘의 직관성과 사용성
단점: Long tail 문제와 데이터 희소성</content>
<sub>NCF Filtering 모델링 및 결과 출력</sub>
<content>Neural Collaborative Filtering 모델링 및 결과 출력.</content>
<sub>SVD 모델 (Singular Value Decomposition)</sub>
<content>사용자-아이템 행렬을 세 개의 행렬로 분해하여 잠재적 특성을 추출하는 방식.</content>
<content>특이값 분해를 사용한 추천 모델은 mXn 행렬에서도 작동하여 사용자가 평가하지 않은 아이템의 평점을 예측할 수 있습니다. 이 모델은 차원 축소로 계산 효율성을 높이고 노이즈를 제거하는 장점이 있지만, 큰 데이터셋에서는 계산 비용이 높아지고 실시간 업데이트가 어렵다는 단점이 있습니다.</content>
<sub>SVD 모델링 및 결과 출력</sub>
<content>모델 성능 평가: 테스트셋을 사용해 모델의 RMSE를 측정.</content>
<sub>SVD 모델링 및 결과 출력</sub>
<content>GridSearchCV로 하이퍼파라미터를 최적화하고, 모델을 재학습한 후 RMSE로 성능을 평가합니다.</content>
<sub>ALS 모델 (Alternating Least Squares)</sub>
<content>행렬 분해(Matrix Factorization)는 추천 시스템에서 사용자-아이템 상호작용 행렬을 두 개의 저차원 행렬로 분해하여 잠재 요인을 학습하는 기법입니다.</content>
<content>교대 최소 제곱법(ALS)은 손실 함수 최적화와 정규화를 통해 대규모 데이터셋에 적합한 알고리즘입니다.</content>
<sub>최종 모델 선정</sub>
<content>모델 선정 이유: ALS (교대 최소 제곱법) 때문.</content>
<content>Implicit Feedback 데이터의 문제점은 사용자의 호불호를 정확히 파악하기 어렵다는 점입니다. 이를 해결하기 위해 선호와 비선호를 binary로 분류하는 방법이 사용됩니다. 또한, Implicit feedback을 대상으로 하는 MF(행렬 분해)에서는 특정 목적함수가 사용됩니다.</content>

<main>결론 및 향후 과제</main>
<sub>프로젝트 요약 및 의의</sub>
<content>여러 추천 알고리즘(CF, NCF, SVD, ALS)을 비교한 결과, Implicit feedback을 이용한 ALS 모델이 최상의 성능을 보였습니다. 이를 통해 사용자에게 맞춤형 웹툰을 추천하여 만족도를 높이고 플랫폼 이용률을 증가시킬 수 있습니다.</content>
<sub>한계점</sub>
<content>정제된 데이터 사용과 rating 값 부재로 인해 사용자 선호도를 정확히 파악하기 어려웠으며, 추천 시스템이 신규 사용자에게 적합하지 않을 수 있음 (Cold Start Problem).</content>
<sub>향후 과제</sub>
<content>명시적 피드백과 genre 컬럼을 활용하여 모델 성능 향상.</content>
