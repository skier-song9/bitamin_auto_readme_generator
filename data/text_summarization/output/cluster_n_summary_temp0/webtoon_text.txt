<main> 프로젝트 배경</main>
<sub>데이터 수집 및 전처리</sub>
<content>프로젝트는 웹툰 관련 데이터를 수집하고 전처리하는 과정을 포함하며, 주요 데이터 소스로는 웹툰 고객 정보, 웹툰 기사, 그리고 사용자 상호작용 데이터가 사용된다.</content>
<main> 프로젝트 목표</main>
<sub>데이터 수집 및 전처리</sub>
<content>649만 개의 데이터를 수집하고, 100개 이상 평가한 유저만을 남겨 데이터의 품질을 높이는 것을 목표로 하며, 이를 통해 피봇 테이블 형식의 데이터를 생성하는 과정이 포함된다.</content>
<main> 데이터 수집 및 전처리</main>
<sub>[데이터 수집]</sub>
<content>사용자와 아이템 간의 상호작용 데이터를 수집하여 추천 시스템의 기초 자료로 활용한다. 이 데이터는 사용자 취향을 이해하고 추천 알고리즘의 성능을 높이는 데 필수적이다.</content>

<sub>[데이터 전처리]</sub>
<content>수집된 데이터는 결측치 처리, 이상치 제거 등의 전처리 과정을 거쳐야 하며, 이를 통해 데이터의 품질을 높이고 추천 알고리즘의 정확성을 향상시킨다.</content>
<main> 데이터 소스 설명</main>
<sub>NCF Filtering 모델</sub>
<content>NCF Filtering 모델은 사용자와 아이템을 각각 embedding vector로 표현하고, 이들 간의 상호작용을 기반으로 학습하여 특정 사용자에게 추천할 아이템의 확률을 높은 순서로 제시하는 방식이다.</content>

<sub>특징</sub>
<content>이 모델은 아이템 간의 복잡한 비선형 관계를 학습할 수 있는 신경망을 활용하며, 사용자와의 상호작용을 통해 추천의 직관성과 사용성을 높인다.</content>

<sub>장점 및 단점</sub>
<content>장점으로는 추천 알고리즘의 직관성과 사용성이 있으며, 단점으로는 Long tail 데이터 문제와 데이터 희소성이 있다.</content>
<main> 데이터 전처리</main>
<sub>[모델 선택 및 학습]</sub>
<content>모델 선택 및 학습 과정에서는 NCF(Neural Collaborative Filtering) 모델을 사용하여 데이터 전처리를 통해 최적의 결과를 도출하는 방법을 설명한다.</content>

<sub>[NCF Filtering]</sub>
<content>NCF Filtering은 데이터 전처리 단계에서 사용자와 아이템 간의 상호작용을 분석하여 모델의 성능을 향상시키는 기법이다.</content>

<sub>[모델링 및 결과 출력]</sub>
<content>모델링 및 결과 출력 단계에서는 전처리된 데이터를 기반으로 NCF 모델을 학습시키고, 그 결과를 시각화하여 성능을 평가하는 방법을 다룬다.</content>

<sub>[Neural Collaborative]</sub>
<content>Neural Collaborative는 신경망을 활용한 협업 필터링 기법으로, 데이터 전처리를 통해 사용자와 아이템의 관계를 효과적으로 모델링하는 데 중점을 둔다.</content>
<main> 모델 선택 및 학습</main>
<sub>SVD(Singular Value Decomposition) 개념</sub>
<content>사용자-아이템 행렬을 세 개의 행렬로 분해하여 사용자와 아이템의 잠재적 특성을 추출하는 방식으로, 차원 축소를 통해 계산 효율성을 높이고 노이즈를 제거하는 장점이 있다. 그러나 큰 데이터셋에서는 계산 비용이 증가하고 실시간 업데이트가 어려운 단점이 있다.</content>

<sub>모델 성능 평가</sub>
<content>모델의 성능은 RMSE(평균 제곱근 오차)를 통해 평가되며, 예측값과 실제값의 차이를 기반으로 계산된다. 라이브러리를 이용하여 SVD 모델을 생성하고 성능을 측정하는 과정이 포함된다.</content>
<main> 모델 비교</main>
<sub>모델 선택 및 학습</sub>
<content>모델 선택 과정에서 SVD와 BITAmir, SVDSingular 모델을 비교하고, 각 모델의 성능을 평가하기 위해 하이퍼파라미터 최적화를 GridSearchCV를 통해 수행하였다.</content>

<sub>모델링 및 결과</sub>
<content>모델링 후, 각 모델의 결과를 출력하여 성능을 비교하였으며, 최적화된 하이퍼파라미터를 적용한 모델의 성능을 재학습을 통해 평가하였다.</content>

<sub>모델 성능 평가</sub>
<content>모델 성능 평가는 RMSE 지표를 사용하여 이루어졌으며, 이를 통해 각 모델의 예측 정확도를 비교하였다.</content>
<main> 최종 모델 선정</main>
<sub>[모델 선정 이유]</sub>
<content>최종 모델로 ALS(Alternating Least Squares)를 선정한 이유는 추천 시스템에서 효과적인 행렬 분해 기법으로, 사용자-아이템 상호작용을 저차원 행렬로 분해하여 잠재 요인을 학습할 수 있기 때문이다. 또한, 대규모 데이터셋에 적합하고 손실 함수 최적화 및 정규화 기능을 통해 성능을 향상시킬 수 있다.</content>
<main> 웹툰 추천 시스템 구현</main>
<sub>[최종 모델 선정]</sub>
<content>ALS 모델을 선정한 이유는 사용자의 호불호를 정확히 파악하기 어려운 implicit feedback 데이터를 효과적으로 처리할 수 있기 때문이다. 이 모델은 대규모 데이터에 적합하며, RMSE 수치가 낮아 최적의 추천 성능을 발휘한다.</content>

<sub>[결론 및 향후 과제]</sub>
<content>여러 추천 알고리즘 모델을 비교 분석한 결과, ALS 모델을 통해 explicit feedback 없이도 사용자에게 맞춤형 웹툰 추천이 가능하다는 결론에 도달했다. 이를 통해 사용자 만족도를 높이고 플랫폼 이용률을 증가시킬 수 있는 가능성을 제시한다.</content>
<main> 기존 사용자</main>
<sub>[최종 모델 선정]</sub>
<content>기존 사용자에 대한 최종 모델로 BITAmir와 ALS(Alternating Least Squares) 모델이 선정되었으며, 이들 모델의 성능을 평가하기 위해 RMSE(평균 제곱근 오차)를 사용하여 결과를 출력하였다.</content>
