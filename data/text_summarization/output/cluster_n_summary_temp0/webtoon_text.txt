<main>프로젝트 소개</main>
<subtitle>프로젝트 배경</subtitle>
<content>이 프로젝트는 정제된 대량의 데이터를 활용하여 모델 선택의 범위를 넓히고, 사용자의 데이터를 효과적으로 사용할 수 있는 주제를 선정하는 것을 목표로 합니다. 또한, 직관적으로 추천 시스템의 작동을 확인하기 위해 국내 최신 데이터를 활용하고자 합니다.</content>

<subtitle>프로젝트 목표</subtitle>
<content>프로젝트의 목표는 사용자 데이터와 웹툰 데이터를 이용해 개인화된 추천 시스템을 개발하는 것입니다. 이를 통해 사용자가 흥미를 느낄 만한 웹툰을 정확히 추천하고, 사용자와 아이템의 데이터에 적합한 알고리즘을 탐색 및 적용하는 것입니다.</content>
<main> 프로젝트 배경</main>
<subtitle>데이터 수집 및 전처리</subtitle>
<content>프로젝트는 웹툰 관련 데이터를 수집하고 전처리하는 과정을 포함하며, 주요 데이터 소스로는 웹툰 고객 정보, 웹툰 기사, 그리고 사용자 상호작용 데이터가 사용된다.</content>
<main> 프로젝트 목표</main>
<subtitle>데이터 수집 및 전처리</subtitle>
<content>649만 개의 데이터를 수집하여 100개 이상 평가한 유저만을 남기는 전처리 과정을 통해 데이터의 품질을 높이는 것이 목표이다. 또한, 피봇 테이블 형식으로 데이터를 생성하여 분석의 효율성을 극대화하고자 한다.</content>
<main> 데이터 수집 및 전처리</main>
<subtitle>데이터 수집</subtitle>
<content>사용자와 아이템 간의 유사도를 측정하기 위해 필요한 데이터는 사용자 구매 및 평가 정보로, 데이터의 크기와 품질이 추천 알고리즘의 성능에 큰 영향을 미친다.</content>

<subtitle>데이터 전처리</subtitle>
<content>데이터 전처리는 사용자 간 유사도(User-Based) 또는 아이템 간 유사도(Item-Based)를 측정하기 위한 과정으로, 데이터의 크기와 정보의 충분성에 따라 적합한 방법을 선택해야 한다.</content>
<main> 데이터 소스 설명</main>
<subtitle>NCF Filtering 모델</subtitle>
<content>NCF Filtering 모델은 사용자와 아이템을 각각 embedding vector로 표현하고, 이들 간의 상호작용을 기반으로 학습하여 특정 사용자에게 추천할 아이템의 확률을 높은 순서로 제시하는 방식이다.</content>

<subtitle>특징</subtitle>
<content>이 모델은 아이템 간의 복잡한 비선형 관계를 학습할 수 있는 신경망을 활용하며, 사용자와의 상호작용을 통해 추천의 직관성과 사용성을 높인다.</content>

<subtitle>장점 및 단점</subtitle>
<content>장점으로는 추천 알고리즘의 직관성과 사용성이 있으며, 단점으로는 Long tail 데이터 문제와 데이터 희소성이 있다.</content>
<main> 데이터 전처리</main>
<subtitle>[모델 선택 및 학습]</subtitle>
<content>모델 선택 및 학습 과정에서는 NCF(Neural Collaborative Filtering) 모델을 사용하여 데이터 전처리를 진행하고, 이를 통해 최적의 모델을 선택하고 학습 결과를 출력하는 방법을 설명한다.</content>

<subtitle>[NCF Filtering]</subtitle>
<content>NCF Filtering은 데이터 전처리 단계에서 사용자와 아이템 간의 상호작용을 기반으로 하여 추천 시스템을 구축하는 기법으로, 효과적인 데이터 전처리를 통해 모델의 성능을 향상시킨다.</content>

<subtitle>[모델링 및 결과 출력]</subtitle>
<content>모델링 및 결과 출력 단계에서는 전처리된 데이터를 활용하여 NCF 모델을 학습시키고, 그 결과를 시각화하여 성능을 평가하는 방법을 다룬다.</content>

<subtitle>[Neural Collaborative]</subtitle>
<content>Neural Collaborative는 신경망 기반의 협업 필터링 기법으로, 데이터 전처리를 통해 얻은 특징을 활용하여 사용자 맞춤형 추천을 제공하는 데 중점을 둔다.</content>
<main> 모델 선택 및 학습</main>
<subtitle>SVD(Singular Value Decomposition) 개념</subtitle>
<content>사용자-아이템 행렬을 세 개의 행렬로 분해하여 사용자와 아이템의 잠재적 특성을 추출하는 방식으로, 차원 축소를 통해 계산 효율성을 높이고 노이즈를 제거하는 장점이 있다. 그러나 큰 데이터셋에서는 계산 비용이 증가하고 실시간 업데이트가 어려운 단점이 있다.</content>

<subtitle>모델 성능 평가</subtitle>
<content>모델의 성능은 RMSE(평균 제곱근 오차)를 통해 평가되며, 예측값과 실제값의 차이를 기반으로 계산된다. 라이브러리를 이용하여 SVD 모델을 생성하고 성능을 측정하는 과정이 포함된다.</content>
<main> 모델 비교</main>
<subtitle>모델 선택 및 학습</subtitle>
<content>모델 선택 과정에서 SVD와 BITAmir 모델을 비교하고, 각 모델의 학습 결과를 분석하였다.</content>

<subtitle>SVDSingular 모델링 및 결과</subtitle>
<content>SVDSingular 모델의 결과를 출력하여 성능을 평가하고, 다른 모델과의 비교를 통해 장단점을 분석하였다.</content>

<subtitle>하이퍼파라미터 최적화</subtitle>
<content>GridSearchCV를 활용하여 각 모델의 하이퍼파라미터를 최적화하고, 이를 통해 모델 성능을 향상시켰다.</content>

<subtitle>모델 재학습 후 성능 평가</subtitle>
<content>모델을 재학습한 후 RMSE를 기준으로 성능을 평가하여, 최적의 모델을 선정하는 데 필요한 정보를 제공하였다.</content>
<main> 최종 모델 선정</main>
<subtitle>모델 선정 이유</subtitle>
<content>최종 모델로 ALS(Alternating Least Squares)를 선정한 이유는 추천 시스템에서 효과적인 행렬 분해 기법으로, 사용자-아이템 상호작용을 저차원 행렬로 분해하여 잠재 요인을 학습할 수 있기 때문이다. 또한, 대규모 데이터셋에 적합하고 손실 함수 최적화 및 정규화 기능을 통해 성능을 향상시킬 수 있다.</content>
<main> 웹툰 추천 시스템 구현</main>
<subtitle>[최종 모델 선정]</subtitle>
<content>ALS 모델을 선정한 이유는 사용자의 호불호를 정확히 파악하기 어려운 implicit feedback 데이터를 효과적으로 처리할 수 있기 때문이다. 이 모델은 대규모 데이터에 적합하며, RMSE 수치가 낮아 최적의 추천 결과를 도출할 수 있다.</content>

<subtitle>[결론 및 향후 과제]</subtitle>
<content>여러 추천 알고리즘 모델을 비교 분석한 결과, ALS 모델을 통해 explicit feedback 없이도 사용자에게 맞춤형 웹툰 추천이 가능하다는 결론에 도달했다. 이를 통해 사용자 만족도를 높이고 플랫폼 이용률을 증가시킬 수 있는 가능성을 제시한다.</content>
<main> 기존 사용자</main>
<subtitle>[최종 모델 선정]</subtitle>
<content>기존 사용자에 대한 최종 모델로 BITAmir와 ALS(Alternating Least Squares) 모델이 선정되었으며, 이 모델을 통해 웹툰 추천 시스템이 구축되었다.</content>

<subtitle>[모델 성능 평가]</subtitle>
<content>모델의 성능은 RMSE(평균 제곱근 오차)를 통해 평가되었으며, 이는 추천 시스템의 정확성을 나타내는 중요한 지표이다.</content>
<main> 신규 사용자</main>
<subtitle>신규 사용자 추천 시스템</subtitle>
<content>신규 사용자를 위한 웹툰 추천 시스템이 필요하며, 기존 사용자 데이터를 기반으로 추천 알고리즘을 개선할 수 있다. 신규 사용자는 이전에 본 웹툰이 없기 때문에, 추천 시스템은 장르나 스토리 타입을 고려하여 적절한 웹툰을 제안해야 한다.</content>
<main> 결론 및 향후 과제</main>
<subtitle>결론</subtitle>
<content>웹툰 추천 시스템은 신규 사용자의 선호도를 기반으로 추천 결과를 제공하며, 입력된 좋아하는 웹툰을 통해 개인화된 추천이 이루어진다.</content>

<subtitle>향후 과제</subtitle>
<content>향후에는 추천 시스템의 정확성을 높이기 위해 RMSE 값을 개선하고, 사용자 피드백을 반영하여 추천 알고리즘을 지속적으로 발전시킬 필요가 있다.</content>
<main> 프로젝트 요약 및 의의</main>
<subtitle>결론 및 향후 과제</subtitle>
<content>프로젝트는 이미 정제된 데이터를 사용하였으나, 사용자와 아이템 간의 상호작용 데이터가 부족하여 사용자 선호도를 정확히 파악하는 데 한계가 있었다. 향후 과제로는 명시적 데이터 수집을 통해 모델 성능을 향상시키고, 아이템 데이터의 장르 정보를 활용하여 모델을 개선할 필요가 있다.</content>
<main> 한계점 및 향후 과제</main>
<subtitle>한계점</subtitle>
<content>프로젝트의 한계점으로는 데이터의 다양성과 품질이 부족하여 추천 시스템의 정확도가 떨어질 수 있다는 점이 지적되었다.</content>

<subtitle>향후 과제</subtitle>
<content>향후 과제로는 데이터 수집의 범위를 넓히고, 알고리즘의 개선을 통해 추천 시스템의 성능을 향상시키는 것이 필요하다.</content>
