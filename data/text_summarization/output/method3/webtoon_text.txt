<main>프로젝트 소개</main>
<sub>프로젝트 배경</sub> <content>정제된 '대량의 데이터'를 사용하여 모델 선택 범위를 넓히고자 함. 사용자의 데이터를 효과적으로 사용할 수 있는 주제 선정. 직관적으로 추천 시스템의 작동을 확인할 수 있도록 가급적 국내 최신 데이터 활용.</content> <page>3</page>
<sub>프로젝트 목표</sub> <content>사용자 데이터와 웹툰 데이터를 이용해 개인화된 추천 시스템 개발. 사용자가 흥미를 느낄 만한 웹툰을 정확히 추천. 사용자와 아이템의 interaction 데이터에 맞는 알고리즘 탐색 및 적용.</content> <page>3</page>

<main>데이터 수집 및 전처리</main>
<sub>데이터 소스 설명</sub> <content>Webtoon_Customers.csv, Webtoon_articles.csv, Webtoon_interaction_train.csv 파일 사용.</content> <page>4</page>
<sub>데이터 전처리</sub> <content>649만 개의 데이터에서 100개 이상 평가한 유저만 남김. 피봇 테이블 형식의 데이터 생성.</content> <page>5</page>

<main>모델 선택 및 학습</main>
<sub>CF (Collaborative Filtering)</sub> <content>사용자 유사도를 측정하는 User-Based와 아이템 간 유사도를 측정하는 Item-Based로 나뉨. 장점: 직관적인 결과, 상품의 정보 없이 추천 가능. 단점: Cold Start Problem, 계산량 많음.</content> <page>7</page>
<sub>NCF (Neural Collaborative Filtering)</sub> <content>사용자와 아이템을 embedding vector로 표현하고, 사용자-아이템 간 interaction을 학습하여 추천. 장점: 추천 알고리즘의 직관성, 사용성. 단점: Long tail, Data Sparsity.</content> <page>9</page>
<sub>SVD (Singular Value Decomposition)</sub> <content>사용자-아이템 행렬을 세 개의 행렬로 분해하여 잠재적 특성을 추출. 장점: 차원 축소로 계산 효율성 증가, 노이즈 제거. 단점: 큰 데이터셋에서 계산 비용 증가, 실시간 업데이트 어려움.</content> <page>11</page>
<sub>ALS (Alternating Least Squares)</sub> <content>사용자-아이템 상호작용 행렬을 두 개의 저차원 행렬로 분해하여 잠재 요인을 학습. 대규모 데이터셋에 적합.</content> <page>14</page>
<sub>최종 모델 선정</sub> <content>ALS 모델 선정 이유로는 Implicit Feedback의 문제점을 해결하고 대규모 데이터에 적합하며, RMSE 수치가 0.1임.</content> <page>15</page>

<main>결론 및 향후 과제</main>
<sub>프로젝트 요약 및 의의</sub> <content>여러 가지 추천 알고리즘 모델 (CF, NCF, SVD, ALS)을 비교 분석하여 최적의 모델 선택. Implicit feedback을 사용한 ALS 모델로 최상의 결과 도출. 추천 시스템을 통해 사용자 만족도와 플랫폼 이용률 증가.</content> <page>20</page>
<sub>한계점 및 향후 과제</sub> <content>정제된 데이터 사용으로 사용자의 선호도를 정확하게 파악하기 어려움. 신규 사용자 추천에 Cold Start Problem 존재. 향후 명시적 feedback 데이터 수집을 통한 모델 성능 향상과 genre 컬럼을 활용한 모델 개선 필요.</content> <page>21</page>
