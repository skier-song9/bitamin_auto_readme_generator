<main>프로젝트 소개</main>
<sub>프로젝트 배경</sub> <content>정제된 '대량의 데이터'를 사용하여 모델 선택 범위를 넓히고자 함. 사용자의 데이터를 효과적으로 사용할 수 있는 주제를 선정하고, 직관적으로 추천 시스템의 작동을 확인할 수 있도록 국내 최신 데이터를 활용.</content>
<sub>프로젝트 목표</sub> <content>사용자 데이터와 웹툰 데이터를 이용해 개인화된 추천 시스템을 개발. 사용자가 흥미를 느낄 만한 웹툰을 정확히 추천하고, 사용자와 아이템의 interaction 데이터에 맞는 알고리즘을 탐색 및 적용.</content>

<main>데이터 수집 및 전처리</main>
<sub>데이터 소스 설명</sub> <content>Webtoon_Customers.csv, Webtoon_articles.csv, Webtoon_interaction_train.csv 등의 데이터를 사용. 다양한 데이터 소스를 통해 웹툰과 사용자 간의 interaction 데이터를 수집.</content>
<sub>데이터 전처리</sub> <content>649만 개의 데이터 중 100개 이상 평가한 유저만 남기고 정제. 피봇 테이블 형식으로 데이터를 생성하여 분석에 적합한 형태로 변환.</content>

<main>모델 선택 및 학습</main>
<sub>Filtering 모델 CF</sub> <content>사용자 유사도를 측정하는 User-Based와 아이템 간 유사도를 측정하는 Item-Based로 나뉨. 데이터 크기와 사용자 정보에 따라 적합한 모델 선택.</content>
<sub>NCF 모델</sub> <content>사용자와 아이템을 embedding vector로 표현하고, 복잡한 비선형 학습을 통해 아이템을 추천. Long tail Data Sparsity 문제 존재.</content>
<sub>SVD 모델</sub> <content>사용자-아이템 행렬을 세 개의 행렬로 분해하여 latent factors를 추출. 차원 축소를 통해 계산 효율성을 증가시키나 큰 데이터셋에서 계산 비용이 높아질 수 있음.</content>
<sub>ALS 모델</sub> <content>교대 최소 제곱법을 이용해 대규모 데이터셋에 적합한 행렬 분해 기법. Implicit feedback 형태의 데이터를 활용하여 모델 성능을 최적화.</content>
<sub>최종 모델 선정</sub> <content>ALS 모델을 최종 선택. Implicit feedback 데이터를 사용해 대규모 데이터셋에서 높은 성능을 나타냄. RMSE 수치 0.01</content>

<main>결론 및 향후 과제</main>
<sub>프로젝트 요약 및 의의</sub> <content>여러 가지 추천 알고리즘을 비교 분석하여 최적의 모델을 선택함으로써 사용자에게 더 나은 추천을 제공. Implicit feedback을 사용하여 ALS 모델을 구축해 최상의 결과 도출.</content>
<sub>향후 과제</sub> <content>Explicit feedback 데이터를 수집해 모델 성능을 향상시키고, genre 컬럼을 활용하여 모델을 개선. Cold Start 문제 해결을 위해 신규 사용자에 대한 추천 시스템 개선 필요.</content>
